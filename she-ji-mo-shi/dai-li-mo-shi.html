代理模式深入分析

代理模式：Proxy，不需要自己做事情，而让其他人帮你代理，最后由你决定。
代理分为三类，静态代理、动态代理，动态代理又分为JDK动态代理和CGLib动态代理。

静态代理源码分析

我们通过一个买房子的例子来看一下什么是静态代理，我们需要买二手房子，但是我们工作很忙，可以通过中介来找房。
我们需要一个中介的接口：

public interface Medium {
    void buy();
}

我们的房地产中介，有一个中介的成员变量，通过构造函数传入进来需要找房子的客户，客户有多种，医生、工程师、企业家，他们只需要实现中介接口，通过买房的中介构造函数传入，由住房中介帮他找房子。

public class HouseAgentProxy {
    private Medium medium;

    public HouseAgentProxy(Medium medium) {
        this.medium = medium;
    }

    public void buy() {
        System.out.println("我是房产中介，帮您找房子");
        this.medium.buy();
        System.out.println("根据您的需求，房子已经找到");

    }
}

然后我们用一个工程师去实现中介接口

public class Engineer implements Medium {
    @Override
    public void buy() {
        System.out.println("我是一名工程师，我平时工作忙，需要找人帮我看房，房子要三室一厅，带暖气");
    }
}

最后我们测试一下：

  public static void main(String[] args) {
        HouseAgentProxy houseAgent = new HouseAgentProxy(new Engineer());
        houseAgent.buy();
    }

输出内容如下：

我是房产中介，帮您找房子
我是一名工程师，我平时工作忙，需要找人帮我看房，房子要三室一厅，带暖气
根据您的需求，房子已经找到

静态代理有个缺点，就是项目中的代理类（XXXProxy）会越来越多

动态代理源码分析

首先我们需要一个实现InvocationHandler接口的类，代码如下：

import java.lang.reflect.InvocationHandler;
import java.lang.reflect.Method;
import java.lang.reflect.Proxy;

public class JDKDynamicProxy implements InvocationHandler {
    //被代理对象
    private Object target;

    public <T> T getInstance(Object target) {
        this.target = target;
        Class<?> clazz = target.getClass();
        return (T) Proxy.newProxyInstance(clazz.getClassLoader(), clazz.getInterfaces(), this);
    }

    @Override
    public Object invoke(Object proxy, Method method, Object[] args) throws Throwable {
        System.out.println("我是房产中介，帮你看房子");
        Object result = method.invoke(this.target, args);
        System.out.println("房子已经找到，有时间过来看");
        return result;
    }
}

我们通过一个main函数测试一下：

 public static void main(String[] args) {
        Medium medium = new JDKDynamicProxy().getInstance(new Engineer());
        medium.buy();
    }

测试结果如下：

我是房产中介，帮你看房子
我是一名工程师，我平时工作忙，需要找人帮我看房，房子要三室一厅，带暖气
房子已经找到，有时间过来看

JDK 原理如下：

1、拿到被代理对象的引用，并且获取到它的所有的接口，反射获取
2、JDK Proxy类重新生成一个新的类、同时新的类要实现被代理类所有实现的所有的接口
3、动态生成Java代码，把新加的业务逻辑方法由一定的逻辑代码去调用（在代码中体现）
4、编译新生成的Java代码.class
5、再重新加载到JVM中运行

以上这个过程就叫字节码重组

CGLib动态代理源码分析

我们需要一个实现MethodInterceptor接口的类，

import net.sf.cglib.proxy.Enhancer;
import net.sf.cglib.proxy.MethodInterceptor;
import net.sf.cglib.proxy.MethodProxy;
import java.lang.reflect.Method;

public class CGLibProxy implements MethodInterceptor {

    public <T> T getInstance(Class<?> clazz) {
        Enhancer enhancer = new Enhancer();
        enhancer.setSuperclass(clazz);
        enhancer.setCallback(this);
        return (T) enhancer.create();
    }

    @Override
    public Object intercept(Object target, Method method, Object[] args, MethodProxy methodProxy) throws Throwable {
        System.out.println("我是房地产中介，帮您看房子");
        Object result = methodProxy.invokeSuper(target, args);
        System.out.println("房子已经找到，有时间来看房子");
        return result;
    }
}

我们通过一个main函数测试一下：

 public static void main(String[] args) {
        Engineer engineer = new CGLibProxy().getInstance(Engineer.class);
        engineer.buy();
    }

输出结果如下：

我是房地产中介，帮您看房子
我是一名工程师，我平时工作忙，需要找人帮我看房，房子要三室一厅，带暖气
房子已经找到，有时间来看房子

我们自己模仿实现一个JDK动态代理：

JDK动态代理模拟实现

首先我们定义一个接口:

import java.lang.reflect.Method;
public interface CustomInvocationHandler {
    public Object invoke(Object proxy, Method method, Object[] args) throws Throwable;
}

我们再创建一个代理类，代理类负责生产代理类对象，具体包括生成字节码和编译.java文件：

import javax.tools.JavaCompiler;
import javax.tools.StandardJavaFileManager;
import javax.tools.ToolProvider;
import java.io.File;
import java.io.FileWriter;
import java.lang.reflect.Constructor;
import java.lang.reflect.Method;


public class CustomProxy {

    public static final String ln = "\r\n";

    public static Object newProxyInstance(CustomClassLoader classLoader, Class<?>[] interfaces, CustomInvocationHandler h) {

        try {
            //1、动态生成源代码.java文件
            String src = generateSrc(interfaces);

            //2、Java文件输出磁盘
            String filePath = CustomProxy.class.getResource("").getPath();
            //System.out.println(filePath);
            File f = new File(filePath + "$Proxy0.java");
            FileWriter fw = new FileWriter(f);
            fw.write(src);
            fw.flush();
            fw.close();

            //3、把生成的.java文件编译成.class文件
            JavaCompiler compiler = ToolProvider.getSystemJavaCompiler();
            StandardJavaFileManager manage = compiler.getStandardFileManager(null, null, null);
            Iterable iterable = manage.getJavaFileObjects(f);

            JavaCompiler.CompilationTask task = compiler.getTask(null, manage, null, null, null, iterable);
            task.call();
            manage.close();

            //4、编译生成的.class文件加载到JVM中来
            Class proxyClass = classLoader.findClass("$Proxy0");
            Constructor c = proxyClass.getConstructor(CustomInvocationHandler.class);
            f.delete();

            //5、返回字节码重组以后的新的代理对象
            return c.newInstance(h);
        } catch (Exception e) {
            e.printStackTrace();
        }
        return null;
    }

    private static String generateSrc(Class<?>[] interfaces) {

        StringBuffer sb = new StringBuffer();
        sb.append("package com.ruoyu.pattern.proxy.customproxy;" + ln);
        sb.append("import com.ruoyu.pattern.proxy.staticproxy.Medium;" + ln);
        sb.append("import java.lang.reflect.Method;" + ln);
        sb.append("public class $Proxy0 implements " + interfaces[0].getName() + "{" + ln);

        sb.append("CustomInvocationHandler h;" + ln);

        sb.append("public $Proxy0(CustomInvocationHandler h) { " + ln);

        sb.append("this.h = h;");

        sb.append("}" + ln);


        for (Method m : interfaces[0].getMethods()) {
            sb.append("public " + m.getReturnType().getName() + " " + m.getName() + "() {" + ln);
            sb.append("try{" + ln);
            sb.append("Method m = " + interfaces[0].getName() + ".class.getMethod(\"" + m.getName() + "\",new Class[]{});" + ln);
            sb.append("this.h.invoke(this,m,null);" + ln);
            sb.append("}catch(Throwable e){" + ln);
            sb.append("e.printStackTrace();" + ln);
            sb.append("}");
            sb.append("}");
        }

        sb.append("}" + ln);

        return sb.toString();
    }

}

我们还需要一个工具类,自定义类加载器，负责加载磁盘上的类

import java.io.ByteArrayOutputStream;
import java.io.File;
import java.io.FileInputStream;
import java.io.IOException;


public class CustomClassLoader extends ClassLoader {

    private File classPathFile;

    public CustomClassLoader() {
        String classPath = CustomClassLoader.class.getResource("").getPath();
        this.classPathFile = new File(classPath);
    }

    @Override
    protected Class<?> findClass(String name) throws ClassNotFoundException {

        String className = CustomClassLoader.class.getPackage().getName() + "." + name;

        if (classPathFile != null) {
            File classFile = new File(classPathFile, name.replaceAll("\\.", "/") + ".class");
            if (classFile.exists()) {
                FileInputStream in = null;
                ByteArrayOutputStream out = null;

                try {
                    in = new FileInputStream(classFile);
                    out = new ByteArrayOutputStream();
                    byte[] buff = new byte[1024];
                    int len;
                    while ((len = in.read(buff)) != -1) {
                        out.write(buff, 0, len);
                    }
                    return defineClass(className, out.toByteArray(), 0, out.size());
                } catch (Exception e) {
                    e.printStackTrace();
                } finally {
                    if (null != in) {
                        try {
                            in.close();
                        } catch (IOException e) {
                            e.printStackTrace();
                        }
                    }

                    if (out != null) {
                        try {
                            out.close();
                        } catch (IOException e) {
                            e.printStackTrace();
                        }
                    }
                }
            }

        }

        return null;

    }
}

我们再创建一个代理类，调用我们刚刚写好的生成代理类的方法。


import java.lang.reflect.Method;


public class CustomHouseAgent implements CustomInvocationHandler {

    //被代理的对象，把引用给保存下来
    private Object target;

    public Object getInstance(Object target) throws Exception{
        this.target = target;

        Class<?> clazz = target.getClass();


        //用来生成一个新的对象（字节码重组来实现）
        return CustomProxy.newProxyInstance(new CustomClassLoader(),clazz.getInterfaces(),this);
    }

    @Override
    public Object invoke(Object proxy, Method method, Object[] args) throws Throwable {
        System.out.println("我是房地产中介，帮助您找房子");
        Object result=method.invoke(this.target,args);
        System.out.println("房子已经找到，有时间来看房子");
        return  result;
    }
}
